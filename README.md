# weather-app-with-testing-git
i make this app just to test git and github for the first time.
this repository will be deleted after i test it.

1 — Repository Setup.
 a- Clone the GitHub repository locally
 b- Connect local project to remote repository
 c- Verify remote connection
 d- Initial commit on main
 e- Push initial commit to GitHub
---------------------------------------
2 — Feature Branch Workflow
  a- Create a new feature branch
  b- Make small feature change
  c- Stage changes
  d- Commit feature
  e- Push feature branch to GitHub
  f- Open Pull Request (PR) on GitHub
  g- Merge Pull Request into main
  h- Pull updated main locally
  i- Delete feature branch (local + remote)
---------------------------------------
3 — Parallel Development Simulation
  a- Create two separate feature branches
  b- Modify the same file differently in each branch
  c- Commit both branches separately
  d- Merge first branch into main
  e- Attempt to merge second branch
  f- Resolve merge conflict manually
  g- Commit resolved conflict
  h- Push updated main
---------------------------------------
4 — Undo Practice
  a- Modify a file without committing → discard changes
  b- Stage a file → unstage it
  c- Make a wrong commit → undo it
  d- Revert a pushed commit safely
  e- Restore a deleted file from history
---------------------------------------
5 — Remote Practice
  a- Create a new branch and push it
  b- Clone repository into a second folder (simulate teammate)
  c- Make change in second clone
  d- Push from second clone
  e- Pull changes in first project
  f- Handle possible pull conflicts
---------------------------------------
6 — Forking Practice
  a- Fork your own repository (or another public repo)
  b- Clone fork
  c- Make changes
  d- Push to fork
  e- Open Pull Request to original repository
---------------------------------------
